我的第一次

1.我对vuex的理解

​ vuex是用来管理全局状态的，就像spring的bean工厂，去管理全局的属性，我在使用组件间传参数时，如果涉及传参多余子到父，或父到子，我就会选择使用vuex。我遇到过一个问题，就是在写文章评论的时候，我开始使用父传子的，article组件把文章id传给子组件CommentList然后渲染每一条评论，又封装了弹出层pop中的评论的组件ComemntPost然后评论成功后会返回一条数据用来更新现在的评论列表，也就是说，现在评论前面需要新增一条数据，这时候问题来了，我得传数据去article父组件，然后再传到CommentList，然后在CommentList进行watch监听，如果有就进行数组的追加。然后我就想这时候使用vuex就好了。

​ 然后之前为了组件复用，我又把每一个评论封装成了comment组件。

​ 评论列表是使用的vant的list组件，使用了懒加载。没加载一次就把数据push进与list组件绑定的数组数据，

然后问题就来了。

​ 我在comment里面点了回复，我想把pop组件框article里面弹出进行评论，pop显示属性show需要传到ComemntList然后传到article，这时候使用vuex就方便了对数据进行统一的管理。

1.为了管理更加方便，我在vuex使用了module模块，开启了命名空间，使用时在store里面新建了module文件夹，然后放了commentModule，按需导出，

2.在store文件夹下的index按需导入，在modules里面使用

3.在.vue单文件里面使用时，和以前一样

import { mapState, mapMutations } from ‘vuex’

4.然后在computed下：

…mapState(‘commentModules’, [‘show’, ‘comment’])

5.在methods下： …mapMutations(‘commentModules’, [‘hideReply’])

2.vant

pop弹出层是懒渲染的，只有当第一次弹出时才渲染数据

3.

文章点赞用了组件，当点击时，发送请求，成功了手动让评论里面的isLiking属性变化，然后给点赞条数加1，点赞的时候开启van-button的等待效果，loading属性，点赞请求发送时，开启，，无论成功还是失败都关闭。取消点赞反之亦然。

4.

文章评论列表使用了list列表，然后每一条评论，又使用了van-cell，通过slot控制样式，在list列表中，使用了懒加载，评论到可视距离底部了，就停止加载，加载时使用trycatch接口。当向上滑动时，使列表没有到可视距离底部了，就开始加载，在加载时，开始loading属性。然后还加了refresh组件，向上滑就加载，加载时发生错误时就在catch里面开启erro属性，并且打开错误提示，当重新加载时使erro为false。

5.

回复层使用popup组件，通过v-model控制弹出和关闭，v-model可以拆开为：value和@input

6.时序问题

1.点击图片van-cell，
2.弹出file表单的点击事件，如果没有图片，直接返回，有图片，也就是this.$refs.imgFileRef.files.length不为0，就弹出图片pop框，然后给pop框的图片的src赋值，
3.直接赋值会发生错误，所以这里的src属性绑定一个数据，然后修改的是这个data里面的这个数据，取消时，关闭popup，清空这个数据，避免下次修改图片，缺取消了后会有数据而导致的popup显示,
4.pop中的图片封装为组件，data里面的数据imgUrl作为props传给子组件，然后进行渲染
5.这里一定得封装组件，不然会发生时序的问题，刚开始，我是直接在组件内写的，方法里面还没有给urlData赋值便去取这个节点的src，导致出错，后面加了settimeout，去取这个值，也使用了process.mextTick(),虽然取得了值，但是还是发生了时序的问题，最后我封装了组件，问题就没有了。

7.keep-alive

<keep-alive> 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。和 <transition> 相似，<keep-alive> 是一个抽象组件：它自身不会渲染一个 DOM 元素，也不会出现在组件的父组件链中。

当组件在 <keep-alive> 内被切换，它的 activated 和 deactivated 这两个生命周期钩子函数将会被对应执行。

在 2.2.0 及其更高版本中，activated 和 deactivated 将会在 <keep-alive> 树内的所有嵌套组件中触发。
