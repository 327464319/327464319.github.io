自定义组件

基本使用（⭐⭐⭐）

  1. 创建自定义组件
    ① 根目录创建一个 components 文件夹
    ② 借助小程序工具，右键创建 component
  2. 在页面中引入自定义组件
    ① 全局引入： 在 app.json 文件中，跟 window 节点同级，声明一个 usingComponents 节点，通过 key value 的形式配置，key 就代表使用的标签名，value 就是自定义组件的路径
    ② 局部引入： 在对应页面的 json 文件中，声明一个 usingComponents 节点，通过 key value 的形式配置，key 就代表使用的标签名，value 就是自定义组件的路径

小程序中对网络请求校验是非常严格的，请求的协议必须是 https 协议
域名需要在后台进行配置（合法域名）
在开发期间可能线上服务器还没有弄好，此时是测试服务器，我们可以通过微信开发工具，详情 -> 本地设置 -> 勾选上不校验合法域名； 项目上线还是必须要在 后台进行配置

导航传参（⭐⭐⭐）

  1. 拼接字符串，queryString 的格式，在 url 地址后面 加上 ?key=value&key1=value1

  2. 目标页面的 onLoad 函数里面就可以接受到参数， 通过 这个函数的 形参

下拉刷新&上拉加载（⭐⭐⭐）

下拉刷新

  1. 在对应页面的 json 文件 开启下拉刷新 enablePullDownRefresh: true

  2. 需要在对应的 js 里面监听下拉刷新的函数 onPullDownRefresh()

  3. 需要在这个函数里面，我们手动调用 wx.stopPullDownRefresh() 关闭下拉刷新效果

上拉加载

  1. 在对应页面 的 json 文件， 配置触底距离（可以不配置，默认是 50px） onReachBottomDistance: 100

  2. 只要触发了触底，就会调用 js 里面 onReachBottom() 这个函数，我们就可以去做分页加载逻辑了

发布（⭐⭐）

小程序发布流程 项目负责人

  1. 借助微信开发工具，找到 上传按钮（工具 -> 上传）

  5. 审核通过之后就可以发布了

事件

如何绑定事件（⭐⭐⭐）

以 bind 开头绑定事件（没有 click 事件与之代替是 tap 事件）

  1. 在 wxml 里面 bind 绑定
  1. 在对应的 js 里面实现这个函数

事件的传参

核心思路：利用自定义属性来传递， 自定义属性遵循 h5 的规范

  1. 绑定事件，设置自定义属性
  1. 在对应的 js 里面实现这个函数,通过 target 属性来拿到值

实现数据的双向绑定

  1. 在 data 里面定义 msg 字段
  1. 把这个 msg 绑定在 input 身上，同时给 input 标签绑定 input 事件
  1. 在事件处理函数里面，获取到用户输入的信息，然后通过 this.setData() 来更新数据

渲染（⭐⭐⭐）

条件渲染

语法：
wx:if 跟我们 vue 中 的 v-if 类似的

语法：
hidden 跟我们 vue 中的 v-show 类似的

列表渲染

我们可以制定别名
wx:for-index 指定索引的别名
wx:for-item 指定元素的别名

wxss （⭐⭐⭐）

① rpx 单位 相对单位，小程序提供的，会把屏幕宽度等分成750份，每一份就是一个rpx， 我们写小程序的时候，单位把px替换成rpx
② @import 进行样式的导入(第三方组件库的使用)
全局的wxss 和 局部的wxss
如果冲突了，并且优先级相同的情况下，采取的是就近原则，取的是局部的

配置项

window节点 （⭐⭐）

  • enablePullDownRefresh 如果开启了之后，会调用js里面 onPullDownRefresh 函数
  • onReachBottomDistance 这个默认值是50px，如果你不配置，也会触发对应的回调函数 onReachBottom

tabBar节点 （⭐⭐）

  • tabBar 如果设置的是position 为top的话，icon图标是不显示的
  • tabBar 里面list的长度必须是 2-5个
  • tabBar 里面配置的 iconPath 或者是 selectedIconPath 不能是网络图片

今天总结

小程序与 PC 开发的区别（⭐⭐）

核心的区别 运行环境不同
pc 运行环境在浏览器
小程序 运行环境在微信
环境不同导致 api 不一致

如何创建一个小程序（⭐⭐⭐）

  1. 注册公众号平台的账户 （年满 18 岁）
  2. 需要拿到 appID 后台 -> 开发 -> 开发设置 -> appID
  3. 不要勾选 云开发

小程序的包结构（⭐⭐⭐）

components 存放组件的
pages 目录： 存放页面的
utils 目录： 存放工具 js 文件的
app.js 小程序的入口文件
app.json 小程序全局配置文件
① pages 存放的是页面的路径 相当于是我们 vue 中的路由
② window 存放的是窗体相关的属性， 例如 navigationBar
app.wxss 小程序全局的样式文件
project.config.json 项目的配置项
sitemap.json 是否被索引 类似于 PC 的 SEO

小程序页面的组成（⭐⭐⭐）

由 4 部分构成

  • .js 文件 小程序页面的逻辑文件（生命周期，事件）
  • .wxml 文件 小程序页面的结构模板 类似于 html 的语法，但是有自己的标签
  • .wxss 文件 小程序页面的样式文件 自己的单位 rpx 专门用来解决移动端适配问题 等分成 750 等分
  • .json 文件 小程序页面的配置文件 （页面的 navigationBar 的样式，或者是注册自定义组件）

宿主环境

了解到什么是宿主环境？ （⭐⭐⭐）
寄生环境 依赖环境

应用程序能够运行的环境
小程序能够运行的环境？ 必须要有微信的 app

通讯（⭐⭐）

通讯主体：
渲染层： wxml 和 wxss
逻辑层： js 代码 xws 脚本语言

运行机制（⭐⭐）

小程序里面的交互都是要依托于微信这个 app 作为中转的
不管是 页面跟 js 交互
或者是 js 调用了第三方服务 api

组件（⭐⭐⭐）

view 组件类似于 div
text 组件 类似于 span
swiper 轮播图 必须要配合 swiper-item 属性 参考文档
scroll-view 滚动组件
① 内容要超过组件宽高
② 设置 scroll-y 属性 或者是 scroll-x 属性
rich-text 解析 html 结构
image 图片组件 mode 设置图片展示的模式 aspectFit
button 组件 按钮
…

api（⭐⭐）

小程序 api 的分类

  • 事件
  • 同步
  • 异步

协同开发（⭐⭐）

管理者两个角色
① 项目开发人员
② 项目体验人员

小程序的启动流程 页面的 启动流程（⭐）
小程序启动流程：
① 下载小程序包
② 解析 app.json 文件找到 pages 里面首页
③ 执行入口文件 app.js 调用 App() 得到实例
④ 渲染内容

页面的启动流程：
① 解析 页面的 json 文件
② 执行对应 .js 文件 Page() 得到实例
③ 找到对应页面进行渲染

1.我对vuex的理解

​ vuex是用来管理全局状态的，就像spring的bean工厂，去管理全局的属性，我在使用组件间传参数时，如果涉及传参多余子到父，或父到子，我就会选择使用vuex。我遇到过一个问题，就是在写文章评论的时候，我开始使用父传子的，article组件把文章id传给子组件CommentList然后渲染每一条评论，又封装了弹出层pop中的评论的组件ComemntPost然后评论成功后会返回一条数据用来更新现在的评论列表，也就是说，现在评论前面需要新增一条数据，这时候问题来了，我得传数据去article父组件，然后再传到CommentList，然后在CommentList进行watch监听，如果有就进行数组的追加。然后我就想这时候使用vuex就好了。

​ 然后之前为了组件复用，我又把每一个评论封装成了comment组件。

​ 评论列表是使用的vant的list组件，使用了懒加载。没加载一次就把数据push进与list组件绑定的数组数据，

然后问题就来了。

​ 我在comment里面点了回复，我想把pop组件框article里面弹出进行评论，pop显示属性show需要传到ComemntList然后传到article，这时候使用vuex就方便了对数据进行统一的管理。

1.为了管理更加方便，我在vuex使用了module模块，开启了命名空间，使用时在store里面新建了module文件夹，然后放了commentModule，按需导出，

2.在store文件夹下的index按需导入，在modules里面使用

3.在.vue单文件里面使用时，和以前一样

import { mapState, mapMutations } from ‘vuex’

4.然后在computed下：

…mapState(‘commentModules’, [‘show’, ‘comment’])

5.在methods下： …mapMutations(‘commentModules’, [‘hideReply’])

2.vant

pop弹出层是懒渲染的，只有当第一次弹出时才渲染数据

3.

文章点赞用了组件，当点击时，发送请求，成功了手动让评论里面的isLiking属性变化，然后给点赞条数加1，点赞的时候开启van-button的等待效果，loading属性，点赞请求发送时，开启，，无论成功还是失败都关闭。取消点赞反之亦然。

4.

文章评论列表使用了list列表，然后每一条评论，又使用了van-cell，通过slot控制样式，在list列表中，使用了懒加载，评论到可视距离底部了，就停止加载，加载时使用trycatch接口。当向上滑动时，使列表没有到可视距离底部了，就开始加载，在加载时，开始loading属性。然后还加了refresh组件，向上滑就加载，加载时发生错误时就在catch里面开启erro属性，并且打开错误提示，当重新加载时使erro为false。

5.

回复层使用popup组件，通过v-model控制弹出和关闭，v-model可以拆开为：value和@input

6.时序问题

1.点击图片van-cell，
2.弹出file表单的点击事件，如果没有图片，直接返回，有图片，也就是this.$refs.imgFileRef.files.length不为0，就弹出图片pop框，然后给pop框的图片的src赋值，
3.直接赋值会发生错误，所以这里的src属性绑定一个数据，然后修改的是这个data里面的这个数据，取消时，关闭popup，清空这个数据，避免下次修改图片，缺取消了后会有数据而导致的popup显示,
4.pop中的图片封装为组件，data里面的数据imgUrl作为props传给子组件，然后进行渲染
5.这里一定得封装组件，不然会发生时序的问题，刚开始，我是直接在组件内写的，方法里面还没有给urlData赋值便去取这个节点的src，导致出错，后面加了settimeout，去取这个值，也使用了process.mextTick(),虽然取得了值，但是还是发生了时序的问题，最后我封装了组件，问题就没有了。

7.keep-alive

<keep-alive> 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。和 <transition> 相似，<keep-alive> 是一个抽象组件：它自身不会渲染一个 DOM 元素，也不会出现在组件的父组件链中。

当组件在 <keep-alive> 内被切换，它的 activated 和 deactivated 这两个生命周期钩子函数将会被对应执行。

在 2.2.0 及其更高版本中，activated 和 deactivated 将会在 <keep-alive> 树内的所有嵌套组件中触发。
