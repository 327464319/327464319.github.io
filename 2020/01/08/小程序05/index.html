小程序-Day05 总结

纯数据（⭐⭐⭐）

就是在 data 里面定义的字段，但是不需要渲染页面这些

用法：

针对小程序有一个优化的处理，调用 this.setData 之后，发现是纯数据的话，就不会去调用起渲染的逻辑

插槽（⭐⭐）

默认用法：

  1. 在自定义组件(my-slot)里面定义插槽 slot
  1. 需要在页面中，给插槽设置内容

多插槽使用：

  1. 开启多插槽
  1. 在自定义组件（my-slot）里面我们利用 slot 标签来设置多个插槽，用 name 属性来指定名称
  1. 页面中，利用 slot 属性来指定内容在哪个插槽里面显示

生命周期（⭐⭐⭐）

组件自身生命周期

组件所在页面的生命周期

通讯方式（⭐⭐⭐）

父传子： 自定义的属性传值

子传父： 通过自定义事件传递，然后在子组件里面通过 this.triggleEvent(‘事件’, 传递的参数)，在父组件的事件回调里面通过 ev.detail 就可以拿到传递过来的值了

兄弟之间： 父传子和子传父
发送数据的这方： 用的是子传父
接收数据的这方： 用的是父传子

获取子组件的实例对象：

behaviors（⭐⭐）

作用: 组件之间共享属性，方法，生命周期的 跟我们 vue 中的 mixin 是类似的

使用：

  1. 定义 behaviors
  1. 使用 behaviors

注意:
① 如果生命周期用冲突, 先执行 behaviors 里面的，再执行组件内部的
② 如果是 data 里面的属性冲突， 就近原则
③ 如果属性没有冲突的话，叠加

npm 模块（⭐⭐⭐）

小程序中支持 npm 模块的

  1. 下载好 npm 模块之后

  2. 需要通过小程序工具， 构建 npm，构建是否成功，你就看根目录上面有没有创建 miniprogram_npm 文件夹

vant Weapp 小程序的组件库
